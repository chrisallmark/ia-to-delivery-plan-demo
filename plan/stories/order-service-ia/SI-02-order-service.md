# Integrate payment gateway for instalments

## Service

**order-service** — REST service managing the full order lifecycle (basket, checkout, payment, fulfilment handoff). Owned by the Payments team.

## Phase

Phase 2 — Domain & Gateway

## Description

Add an `InstalmentGatewayClient` class to `src/payments/gateway.py` that wraps the existing Stripe Python SDK to create payment intents with instalment schedules. The client must read `STRIPE_SECRET_KEY` from environment variables (already injected by the existing pattern). Depends on SI-01 for the domain types passed to and returned from the gateway call. Must not be started until Stripe Instalments is confirmed enabled on the account.

**Target file:** `src/payments/gateway.py`. The existing file already wraps Stripe for standard payment intents; add `InstalmentGatewayClient` as a new class (or extend the existing wrapper class) with a `create_instalment_intent(plan: InstalmentPlan) -> str` method that returns the Stripe payment intent ID.

**Credentials:** `STRIPE_SECRET_KEY` is already read from environment variables in `gateway.py`; follow the existing pattern. Do not introduce a new config mechanism.

**Tests:** Add a `test_gateway.py` (or extend the existing one) that mocks the Stripe SDK and verifies the correct parameters are passed to `stripe.PaymentIntent.create`.

**Prerequisite gate:** Stripe Instalments must be confirmed enabled on the account before this SI is tested end-to-end. Unit tests with mocked SDK can proceed without it.
