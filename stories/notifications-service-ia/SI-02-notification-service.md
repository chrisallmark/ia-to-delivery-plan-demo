# Implement FCM push provider

## Service

**notification-service** — REST service that dispatches transactional notifications across email, SMS, and (soon) push channels. Node.js 20 / TypeScript 5 / Fastify 4 / PostgreSQL 15.

## Phase

Phase 2 — Provider & Preferences

## Description

Implement `FirebasePushProvider` satisfying the `PushProvider` interface established in SI-01. The provider must use the Firebase Admin SDK to dispatch push notifications. Firebase Admin SDK credentials must be read from Vault (environment variables) — do not hard-code or commit them. This SI replaces the no-op stub from SI-01 with the real implementation. Does not depend on SI-03 and can be developed in parallel with it.

## Implementation hint

> ⚠️ This implementation hint is generated by Claude Code from the service's
> CLAUDE.md. It is a starting point, not a specification. Verify against the
> current codebase before starting work.

**Location:** `src/channels/push/FirebasePushProvider.ts` — implement the class within the `src/channels/push/` folder created in SI-01. Export it from `src/channels/push/index.ts` and register it in the factory.

**Firebase Admin SDK:** The CLAUDE.md notes Firebase Admin SDK is not yet integrated. Add `firebase-admin` as a dependency (`pnpm add firebase-admin`). Initialise the SDK once at service startup using credentials from an environment variable (e.g. `FIREBASE_SERVICE_ACCOUNT_JSON` as a JSON string, or individual `FIREBASE_PROJECT_ID` / `FIREBASE_CLIENT_EMAIL` / `FIREBASE_PRIVATE_KEY` vars — confirm with the Platform team and Vault admin which pattern Vault uses for this service).

**Credential rotation:** The IA notes credentials must be rotated per environment. Ensure the SDK initialisation reads from environment at startup, not at module load time, so credential rotation can be applied via a service restart without a code change.

**Tests:** Mock the Firebase Admin SDK in Vitest tests (`vi.mock`). Test happy path (message sent, FCM message ID returned) and error path (Firebase error surfaced with appropriate HTTP status).
