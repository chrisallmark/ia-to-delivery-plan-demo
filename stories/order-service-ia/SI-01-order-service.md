# Add instalment plan data model

## Service

**order-service** — REST service managing the full order lifecycle — basket, checkout, payment and fulfilment handoff. Python 3.12 / FastAPI / asyncpg / PostgreSQL 15.

## Phase

Phase 1 — Foundation

## Description

Introduce `instalment_plans` and `instalment_schedules` tables to the PostgreSQL database. Add corresponding domain models (dataclasses in `src/orders/models.py`) and a repository class (in `src/orders/repository.py` or a new `src/payments/repository.py`) exposing async CRUD helpers for both tables. This SI is the schema and domain layer only — no routes or gateway calls. All downstream SIs (SI-02 through SI-05) depend on this being merged and deployed before they begin.
