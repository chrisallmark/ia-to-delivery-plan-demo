# Integrate payment gateway for instalments

## Service

**order-service** — REST service managing the full order lifecycle — basket, checkout, payment and fulfilment handoff. Python 3.12 / FastAPI / asyncpg / PostgreSQL 15 / Stripe SDK 9.x.

## Phase

Phase 2 — Gateway & Calculator

## Description

Add an `InstalmentGatewayClient` class to `payments/gateway.py` that wraps the existing Stripe SDK to create payment intents with instalment schedules. The client must read `STRIPE_SECRET_KEY` from the environment (already injected — do not hard-code). The method signature should accept `order_id`, `amount`, `currency`, and the schedule returned by `InstalmentCalculator` (SI-03). This SI delivers the gateway abstraction only; it is composed into the checkout endpoint in SI-04. Requires SI-01's data model to be in place to type-check against `InstalmentPlan`.

## Implementation hint

> CLAUDE.md. It is a starting point, not a specification. Verify against the
> current codebase before starting work.

**Location:** `src/payments/gateway.py` — add a `create_instalment_intent` method to the existing Stripe wrapper class (or a new `InstalmentGatewayClient` if the existing class is already large).

**Stripe Instalments:** this is a distinct Stripe product feature (not standard PaymentIntents). Confirm the Stripe account has Instalments enabled before writing code against it — the API will return errors in test mode until it is activated.

**Credentials:** `STRIPE_SECRET_KEY` is already injected as an environment variable per the CLAUDE.md conventions. No new secret configuration is needed, but the Stripe Instalments feature may require additional configuration in the Stripe dashboard.

**Tests:** Mock the Stripe SDK in unit tests (`pytest` + `unittest.mock`). Test the happy path (intent created, reference returned) and the error path (Stripe API error surfaced correctly).
