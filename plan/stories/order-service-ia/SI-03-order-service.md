# Add instalment calculation service

## Service

**order-service** — REST service managing the full order lifecycle (basket, checkout, payment, fulfilment handoff). Owned by the Payments team.

## Phase

Phase 2 — Domain & Gateway

## Description

Implement `InstalmentCalculator` in `src/payments/calculator.py`. The calculator derives a repayment schedule from order total, term length (months), and APR. Output must match the figures the Stripe gateway returns to the nearest penny — rounding edge cases must be covered by unit tests. This is pure domain logic with no external calls; it may be developed and tested fully independently of SI-02, only requiring the `InstalmentSchedule` domain type from SI-01.

**Method signature:** `calculate(order_total: Decimal, term_months: int, apr: Decimal) -> list[InstalmentSchedule]`. Use `decimal.Decimal` throughout — do not use `float` for monetary arithmetic.

**Rounding:** Python's `Decimal` with `ROUND_HALF_UP` and `quantize(Decimal('0.01'))` is the standard approach. Ensure the sum of all instalment amounts equals `order_total` exactly — distribute any remainder cent to the final instalment.

**Tests:** Co-locate `test_calculator.py` in the same directory. Cover: equal division, non-divisible totals (penny remainder), minimum APR, maximum APR, single-instalment degenerate case.
