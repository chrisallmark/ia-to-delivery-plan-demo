# Add delivery status webhook

## Service

**notification-service** — REST service dispatching transactional notifications across email, SMS, and (soon) push channels. Owned by the Platform team.

## Phase

Phase 4 — Status Webhook

## Description

Add `POST /notifications/push/status` to `src/routes/` to receive FCM delivery receipts. The endpoint must: parse and validate the FCM status payload; update the delivery record for the relevant notification (set status to `delivered` or `failed`); emit a domain event via `src/events/emit.ts` using the existing event schema in `src/events/schema.ts`. The event schema must match the email/SMS domain event format. Depends on SI-04 for delivery records to exist.

## Implementation hint

> ⚠️ This implementation hint is generated by Claude Code from the service's
> CLAUDE.md and source files. It is a starting point, not a specification.
> Verify against the current codebase before starting work.

**Target file:** `src/routes/` — extend `pushRoutes.ts` (from SI-04) with the status endpoint, or add a separate `pushStatusRoutes.ts` depending on route organisation conventions in the existing codebase.

**FCM receipt format:** FCM delivery receipts are sent as HTTP POST callbacks. Validate the incoming payload structure (notification ID, status: `delivered` | `failed`, timestamp). Return `400` for malformed payloads.

**Delivery record update:** Identify the notification by its FCM message ID (stored when dispatched in SI-04). Update the status and timestamp in the database using a new query helper in `src/db/`.

**Domain event:** After the DB write, call `emit()` from `src/events/emit.ts` with an event conforming to `src/events/schema.ts`. The event type must match the pattern used by email and SMS status events — do not invent a new schema shape. Verify the expected field names (e.g. `notificationId`, `channel`, `status`, `timestamp`) against the existing schema before implementing.

**Tests:** Cover: delivered receipt updates record and emits event, failed receipt updates record and emits event, malformed payload returns 400, unknown notification ID returns 404.
