# Persist notification preferences

## Service

**notification-service** — REST service dispatching transactional notifications across email, SMS, and (soon) push channels. Owned by the Platform team.

## Phase

Phase 2 — Provider & Preferences

## Description

Add a `push_preferences` table to the PostgreSQL database to store device tokens and per-user opt-in state for push notifications. Wire the new table into the existing preferences API in `src/preferences/`. A new numbered migration file is required. Depends on SI-01 for the foundational push channel structure but is otherwise independent of SI-02. Both SI-02 and SI-03 must be merged before SI-04 can be built.
